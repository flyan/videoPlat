# 快速开始 - HTTPS 访问

## ✅ HTTPS 已配置完成

VideoPlat 现在支持 HTTPS 访问，解决了 IP 地址访问时无法使用摄像头/麦克风的问题。

## 🚀 如何访问

### 1. 使用 HTTPS 访问（推荐）

```
https://192.168.10.9
```

### 2. 首次访问 - 信任证书

由于使用的是自签名证书，首次访问会看到安全警告：

#### Chrome/Edge 浏览器
1. 看到"您的连接不是私密连接"警告
2. 点击"高级"
3. 点击"继续前往 192.168.10.9（不安全）"

#### Firefox 浏览器
1. 看到"警告：潜在的安全风险"
2. 点击"高级"
3. 点击"接受风险并继续"

### 3. 开始使用

信任证书后，即可正常使用所有功能：
- ✅ 登录
- ✅ 创建会议
- ✅ 使用摄像头/麦克风
- ✅ 屏幕共享
- ✅ 会议录制

## 🔄 HTTP 访问

访问 `http://192.168.10.9` 会自动重定向到 HTTPS。

## 📱 移动设备访问

移动设备也需要信任证书：

### iOS
1. 访问 `https://192.168.10.9`
2. 点击"显示详细信息" → "访问此网站"
3. 设置 → 通用 → 关于本机 → 证书信任设置 → 启用完全信任

### Android
1. 访问 `https://192.168.10.9`
2. 点击"高级" → "继续访问"

## ❓ 常见问题

### Q: 为什么需要 HTTPS？
A: 浏览器安全策略要求访问摄像头/麦克风必须在 HTTPS 或 localhost 环境下。

### Q: 自签名证书安全吗？
A: 自签名证书提供加密传输，适合开发和内网环境。生产环境建议使用受信任的 CA 证书。

### Q: 每次都要点击"继续访问"吗？
A: 不需要。信任一次后，浏览器会记住该证书。

### Q: 如何永久信任证书？
A: 参考 `HTTPS配置说明.md` 中的详细步骤。
